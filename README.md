# 概要
* はじめてPythonプログラムを作って、webアプリとしてブラウザ公開するための練習で作成。
* アプリの内容は、株の取得価格と取得株数を入力するとお決まりの利確ポイント（+1~5%）を算出するもの。超単純なテストアプリ。
* 環境は、Anaconda環境＋JupyterLabで書いた上記プログラムをGitHubにアップロードし、Streamlitで公開する（言葉や概念の理解はあってる？）練習用です。

# 勉強したこと（備忘録）
書くことないから勉強したこと記録しとく。
* READMEファイルとは  
アップするこのpyプログラム自体の**説明書**。  
最初に読んでねというやつ。いろんなアプリインストールしたときに入ってるお決まりのやつ。フォントとかインストールすると「DLありがと～このフォントは商用利用で・・・」みたいなのがtxtファイルで入ってるやつ。別にtxtファイルでもいいが通常は拡張子「md」が使われる。
* ファイルの拡張子 .mdとは  
Markdown（マークダウン） の略で、txtファイルより圧倒的に見やすくて便利なため今ではこの拡張子で書くことが習慣化されている。  
「書きやすくて読みやすいプレーンテキスト」を目指して開発された軽量マークアップ言語。
特徴: シャープ記号 (#) やアスタリスク (*) のようなシンプルな記号を使って、見出し、箇条書き、太字、リンクといった装飾や構造を表現できます。
レンダリング: GitHubなどのプラットフォームや多くのテキストエディタは、このMarkdown記法で書かれたテキストを自動的に整形し、見栄えの良いHTML形式（ウェブページ）として表示する機能を持っています。  
＜装飾の例＞
  * 「\#」＋ 半角スペース　見出し。\#の数が多いほど小さくなる。h1~h6  
  * 「\*」＋ 半角スペース　順番なしリスト（箇条書き）。入れ子するときはスペースを2つ入れる。  
  * 改行　半角スペースを2つ入れてから改行  
  * 太字　\*\*文字\*\*　で**太字**になる。
